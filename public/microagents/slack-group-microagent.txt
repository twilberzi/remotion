Jagmohan Singh  [5:24 PM]
Team, If you're encountering errors during the Micro-App creation process, the issue might be related to your npm registry configuration.
Quick Fix Steps:
1. Check Your Current Registry
First, verify which npm registry you're currently using:
npm config get registry
2. Compare with the Standard Command
The standard Micro-App creation command uses this registry:

npx --registry=https://artifacts.zi-int.com/artifactory/api/npm/zi-npm-virtual/@zoominfo-chat/create-micro-app <your-micro-app-name>
3. Match Your Registry Configuration
If your registry doesn't match, you need to update the command to use your specific registry URL from your .npmrc file configuration.
Example Solution:
In my case, my npm registry is configured as:

registry=https://zoominfo.jfrog.io/artifactory/api/npm/virtual-npm/

So I use this modified command:
BASH
npx --registry=https://zoominfo.jfrog.io/artifactory/api/npm/virtual-npm/@zoominfo-chat/create-micro-app <your-micro-app-name>

Tip to reduce usage cost in Cline
As we continue working enthusiastically on AI projects, here’s one way to save costs.

Cline usage costs grow exponentially with longer tasks. To lower costs, restart tasks before the context window gets too large.
For example: when building a UI, once the context window reaches around 60%(screen shot below), type “summarize the chat and start a new task.” Cline will prompt “Start New Task with Context.” Select it, and the new task resets to $0.
Note: Use this method only for tasks that can be restarted easily, such as UI building.

@channel Hi team! Over the last month we've been building some powerful new capabilities to expand the Microapps platform and we're excited to announce them today. A special thank you to @danny.myers and @jered.little for their incredible work in this release.

	•	Development Mode: this allows you to run develop your application side-by-side with your production one in the same workspace, with access to the same tools/data.
	•	Deep Linking: when set up, you will be able to store "state" in the address bar.
	•	Snowflake Data Store (EA): this gives your application access to write custom queries + manage your own data in Snowflake. This capability is now in early access testing and we're excited to hear your feedback. Send us a message in this channel if you'd like access.
	•	Postgres Data Store (EA): this gives your application access to a database where you can store microapp specific data without resorting to weird tooling hacks. This capability is now in early access testing and we're excited to hear your feedback. Send us a message in this channel if you'd like access.
	•	User List: you can get a list of the currently available ZI Chat users with apiClient.getUsers().

npx @zoominfo-chat/create-micro-app <your-micro-app-name>
npm install
npm run dev

Follow the steps here and then re-deploy your application.
npm update @zoominfo-chat/micro-app-core

 Upcoming Breaking Changes

 Required Staging Migration
